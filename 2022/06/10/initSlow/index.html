记录一次项目启动慢的解决方法

起因

今天切换分支解决了一个小 bug，十分轻松。提交代码后切换回主开发分支，运行项目时发现特别慢。刚开始以为是缓存的问题，于是重新运行项目，结果还是很慢。

尝试解决问题

刚刚在fixbug分支运行项目时，提示缺少依赖，所以有过一次下载依赖的操作，我怀疑是依赖的问题。看了看公司上传私服的包，没看出问题，而且同事更新依赖后运行正常，所以 pass 掉依赖问题。

项目是在 Spring Boot 刚启动不久后卡住不动的，有可能是加载的 bean 太多，内存不够用，于是修改了 idea 的虚拟机设置，发现还是不行，甚至改了编译线程数也不行（废话，日志都出来了，编译过程早就过去了）

看日志，显示初始化数据后就停留了 10 分钟，会不会是数据库的问题？

我启动的是本地项目，SQLyog访问正常，数据库配置没改动，那么也不是数据库的问题。至于下一行的初始化线程池服务，也同数据库一样，没改过。

那么，在两者间到底发生了什么？

改动项目的日志输出级别为 INFO，大部分的日志都能数据（涉及公司项目机构，就不贴代码了）。

日志输出仍然是卡住初始数据库后，初始化线程池服务前也没有任何日志，这说明了并不是 Spring Boot 初始化的问题。那么，除去 JVM、Spring Boot 后，只剩下一个目标了：idea。于是百度搜索 idea 初始化项目慢的问题，结果让找到一个方法：先用 Maven clean 一遍，然后清掉 idea 的所有断点。照做一遍，恢复了正常速度！！！！

先前有进行过 clean 操作，那么很明显就是 idea 的断点卡住了 Spring Boot 的初始化。不过很奇怪，我明明已经关闭了 Debug 模式，不知为何还会卡住，这可能是 idea 的 bug 吧。

对于该功能的探索，我后来看到了一篇文章，讲的比较详细：https://juejin.cn/post/7131956856771051557#heading-0
